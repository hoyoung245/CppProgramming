```
알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?
```
알고리즘은 문제를 해결하는 방법을 정하는 과정이고, 코딩은 그 방법을 프로그램으로 구현하는 것이다. 알고리즘이 잘못되면 코딩을 잘해도 문제를 해결할 수 없기 때문에 알고리즘 설계가 더 중요하다.
```
클래스와 객체가 무엇인지 설명하시오.
```
클래스는 객체를 만들기 위한 설계도이며, 변수와 함수를 포함한다.
객체는 클래스에서 만들어진 실제 데이터나 프로그램 속의 실체를 의미한다.
즉, 클래스가 설계도라면 객체는 그 설계도로 만들어진 실제 결과물이다.
```
절차지향 프로그래밍 언어와 객체지향 프로그래밍 언어의 차이를 조사하라.
```
절차지향 프로그래밍 언어는 함수나 절차를 중심으로 코드를 작성하는 방식이고, 객체지향 프로그래밍 언어는 클래스와 객체를 중심으로 데이터를 처리하고 기능을 구현하는 방식이다.
절차지향은 어떻게 처리할지를, 객체지향은 무엇을 처리할지를 중심으로 설계한다.
```
라이브러리가 무엇인지 자세히 설명하시오.
```
라이브러리는 프로그래밍에서 자주 사용하는 기능들을 미리 만들어 놓은 코드 모음이다.
개발자는 라이브러리를 사용해 처음부터 코드를 작성하지 않고도 기능을 쉽게 구현할 수 있으며, 개발 속도를 높이고 오류를 줄일 수 있다.
```
라이브러리가 목적파일 형태로 존재하는 이유를 설명하라.
```
라이브러리는 목적 파일 형태로 존재하여 재사용이 쉽고, 컴파일 시간을 줄이며, 소스 코드를 보호할 수 있고, 프로그램과 연결하기 편리하기 때문이다.
```
링크과정이 필요한 이유를 설명하라.
```
링크 과정은 여러 목적 파일과 라이브러리를 하나로 합쳐 실행 가능한 파일을 만들고, 각 함수와 변수에 메모리 주소를 지정하기 위해 필요하다.
