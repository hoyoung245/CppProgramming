# 실습과제 2

# 문제점
반복문 안에서 Circle 객체를 new로 동적 생성하지만 delete로 메모리를 해제하지 않음.
이 때문에 프로그램 실행 중에 메모리가 계속 차지되어 메모리 누수(memory leak)가 발생한다.

# 해결 방법
동적 생성한 객체는 사용 후 delete로 반드시 해제해야 한다.
즉, 반복문 안에서 객체 사용 후 delete p;를 추가하면 메모리 누수를 방지할 수 있다.


# 실습과제 3

# 문제점
동적 배열 pArray를 생성한 후 포인터를 직접 증가시키며 배열을 순회하면, 배열의 시작 주소를 잃어버림.
그 결과 delete[] pArray;가 올바른 주소를 받지 못해 메모리 누수가 발생한다.

# 해결 방법
배열 순회 시 원본 포인터를 변경하지 않는다.
포인터 연산을 쓸 경우 임시 포인터를 사용하거나 배열 인덱스를 이용한다.
